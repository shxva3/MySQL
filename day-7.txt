TCL(Transaction Control Language)
---------------------------------
The Process of combing all related operations into a single unit and executuing them
according to the rule "either all operations should be succeed or none should be succeed"
is called Transaction management

A transaction is considered a single unit of work and fallows the principle
"either all or none"
Example: Funds transfer.
consider a money transfer process between two accounts
1.Debit amount from the senders account
2.Credit amount into the receivers account.

Both operation should be executed as single transaction

if the amount is debit from the senders account successfully. But crediting the recovers 
account fails.it may lead to the data inconsistency problems.

They are two types of Transaction
1.Local Transactions
2.Global Transactions

1.Local Transaction: In Local all operations  of transactions are executed in the same database
Example
-------
In local Transaction from one account to another account where both accounts belongs to the same bank.

2.Global Transaction 
-------------------
In Global all operations  of transactions are executed in the different database
Example
-------
Funds transfer from one account  to another account where both accounts belongs to the different bank

Any tansaction should be follows acid Properties
ACID Properties
--------------
Atomicity: All or nothing
Consistency: the databases remains in a valid states 
Isolation: Transaction do not interface with each other
Durability: Committed changes are permanent

TCl Commands:
1,Commit
2.Rollback
3.Save Point

What is locking
--------------
locking is mechanism used by the database to control access data.
 
it  ensure that multiple users access the same database simultaneously the database maintains
consistency,integrity,and prevent conflicts.

1.Shared lock
--------------
A shared lock allows a transaction to read data.
Multiple transactions can hold a shared lock  on the same row.
No transaction can modify the row until all shared locks are released

Characteristics
Read Only
Multiple user can read simultaneously
Blocks updates and delete

2.Exclusive lock
----------------
An Exclusive lock allows a transaction to modify data
while on Exclusive lock exists.
No other transaction can read using a locking read or modify that row.
Only the transaction holding the lock can update and delete the data

Characteristics
used for Insert, update, Delete 
only one Exclusive lock at time
Blocks both reads(locking reads) and writes 

3.INTENT Lock
-------------
An Intent lock is a table-level lock that indicates a transaction to place row -level
locks
It helps My sql  quickly determine whether a table-level lock conflicts with existing row locks.
users do no usually create intent locks manually. InnoB manages them automatically.

Real -Life Example
Imagine a library
Shared lock - Many students can read the same book at the same time.
Exclusive lock - one student borrows the book, so  nobody else can read or borrow it until it is
returned.

Intent Lock - the librarian marks that someone is about to read or borrow books from a shelf,
helping coordinate access without checking every book individually
